# Conducting a Security Audit

## Objective

The goal is to conduct a comprehensive cybersecurity audit to evaluate the effectiveness of current security controls, identify vulnerabilities and compliance gaps, and provide actionable recommendations to strengthen the organization’s overall security posture and risk management practices.



### Skills Learned

- Advanced understanding of risk assessment and threat analysis
- Proficiency in analyzing and identifying security gaps and vulnerabilities
- Evaluating security controls and policies
- Enhanced knowledge of access control and privilege management review
- Development of critical thinking and problem-solving skills in cybersecurity.
- Documentation, reporting, and audit evidence collection
- Communicating technical findings to non-technical stakeholders
- Developing remediation and risk mitigation recommendations



### Tools Used

1. Vulnerability Assessment & Scanning
- Nessus – Network vulnerability scanner


2. Network & Traffic Analysis
- Wireshark – Packet capture and traffic analysis
- tcpdump – Command-line network traffic capture


3. Configuration & Complaince Auditing
- Lynis – Linux/Unix system auditing tool


4. Web & Application Security Testing
- OWASP ZAP – Open-source web vulnerability scanner


5. Log Analysis & SIEM Tools
- Splunk – Log aggregation and analysis


6. Password & Access Control Auditing
- Hydra / John the Ripper / Hashcat – Password strength and policy checks
- Netwrix Auditor – User activity and access auditing


7. Reporting & Documentation
- Excel / Google Sheets – Audit evidence and tracking
